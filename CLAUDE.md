# CLAUDE.md — MindAttic.Ideas (how to work here)

MindAttic.Ideas is a single-deployment Blazor CMS: upload a `.idea` zip and it goes live with no
redeploy. See `README.md` for the build/run details and the living feature spec.

## Codex — the canonical documentation standard

This repo follows the MindAttic **Codex** standard. The canon lives in `docs/`:

| File | Layer | What it is |
|---|---|---|
| `docs/BIBLE.md` | L0 | Source of truth — what the project IS, is NOT, its architecture, and its Laws. Stable IDs `{#MAI-§N}` / `{#MAI-LAW-n}`. |
| `docs/AMENDMENTS.md` | L1 | Append-only change log (`MAI-A1..A18`). **An amendment wins over the bible.** Never rewrite one; supersede with a new ID. |
| `docs/USER_STORIES.md` | L2 | Test-cited stories (`MAI-US-<Epic><n>`). Every ✅ names its NUnit test. |
| `docs/rfc/*.md` | — | Design notes; graduate into L0/L2 then mark superseded. RFC 0001 = the unified page-grammar plan. |
| `docs/BIBLE.digest.md` | — | GENERATED by `tools/codex.ps1`; injected at session start. Never hand-edit. |
| `../MindAttic.HouseRules.md` | inherited | Org-wide laws (HOUSE-LAW-1..9). The bible §5 inherits these by reference. |

Historical: `docs/FOUNDATION_ADR.md` (the frozen Legion deliberation) and `docs/FOUNDATION_AMENDMENTS.md`
are preserved for existing links; current truth is `BIBLE.md` + `AMENDMENTS.md`.

### Rules of engagement
- **Single home per fact.** A fact lives in one layer; everything else links to it by `{#id}`.
- **Status legend:** ✅ done (a test/build proves it) · 🟡 partial · ⬜ planned · 🗑️ cut · living.
  Mark ✅ only when verified (HOUSE-LAW-8).
- **Current vocabulary** is the four kinds **Page · Widget · Theme · Control** under `IdeaBase`
  (amendment A18 renamed the composable-UI kind to *Widget*; older docs say Component/Plugin).
- **Whole-number versioning only** (`V1`, `V2`, …) — never SemVer (HOUSE-LAW-1).
- When the canon changes: edit `BIBLE.md`/`USER_STORIES.md`, append an amendment if a frozen decision
  changes, then run `tools/codex.ps1 digest` and `tools/codex.ps1 doctor`.

### Tooling
```pwsh
powershell -File tools/codex.ps1 digest   # regenerate docs/BIBLE.digest.md
powershell -File tools/codex.ps1 doctor   # validate the canon (must pass)
```
The `.claude/hooks/inject-digest.ps1` SessionStart hook (registered in `.claude/settings.json`) injects
the digest as authoritative context at the start of every session.

### Build & test
```pwsh
dotnet build MindAttic.Ideas.slnx -c Debug
dotnet test src/MindAttic.Ideas.Tests/MindAttic.Ideas.Tests.csproj
```
A feature is done only with a clean build, green tests, and (for user-facing work) an e2e/lifecycle
assertion. Do not modify application/source code as part of a docs task.
